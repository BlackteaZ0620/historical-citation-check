# historical-citation-check
这是一个面向历史学论文写作的skill，用于检查和整理论文中的脚注、尾注与文献注释。它会按照历史研究常见注释体例，帮助用户识别注释中的格式问题、字段缺漏、标点问题、外文斜体问题和页码标注问题。
## 安装方式
Windows PowerShell 一行安装
用户只需要复制下面这一行到 Windows PowerShell：
```
irm https://raw.githubusercontent.com/BlackteaZ0620/historical-citation-check/main/install.ps1 | iex
```
macOS / Linux
```
git clone https://github.com/BlackteaZ0620/historical-citation-check.git ~/.codex/skills/historical-citation-check
```
## 主要功能
使用本 skill 后，工具会读取用户提供的 Word 文档，并尽量保留原文中的斜体、空格、中英文标点和脚注内容。检测应基于 Word 原始字符和 run 级格式区间，而不是基于转换后 Markdown 中额外添加的星号。默认会生成一份文件：
citation-errors.md
记录论文注释中发现的问题，包括缺少字段、顺序错误、标点不规范、外文书名或期刊名未斜体、页码格式不一致等。
如果用户明确要求 JSON 辅助数据，工具可以按具体文献类型另行生成结构化文件。不同文献类型的字段不同，不应把著作、期刊、报纸、档案、古籍和外文文献全部塞进同一个固定 JSON 模板。
## 适用场景
本 skill 适合用于：
历史学课程论文、毕业论文、学位论文的注释检查。
中文著作、译著、期刊论文、报纸文章、文集析出文献、古籍、档案材料和外文文献的注释校对。
投稿前检查论文注释是否符合历史研究类注释格式。
将 Word 文档中的脚注、尾注转换为 Markdown 错误报告。
## 注意事项
使用前请务必确认自己的投稿论文、课程论文或学位论文要求采用历史研究注释格式。不同学校、期刊、出版社可能有自己的注释规范，本 skill 不能替代投稿指南或导师要求。

如果原始注释本身缺少必要信息，例如缺少期刊名、刊号、卷号、期号、页码、出版社、出版时间、档案编号或馆藏地点，本 skill 不能自动补全这些信息。它只能指出缺失项，不能凭空生成真实的文献信息。

本 skill 的主要作用是检查和列出注释错误，并生成 Markdown 错误报告。它不会自动替用户改写 Word 文档中的全部注释。用户如果希望继续修改论文，可以根据错误报告，通过自然语言继续命令 Codex、其他编程工具或人工执行修改。

外文书名和期刊名的斜体、英文逗号、中文逗号、空格、页码符号等细节会影响判断结果。请尽量提供原始 Word 文档，而不是经过多次复制粘贴后格式已经丢失的纯文本。

如果 Word 把一个斜体词拆成多个 run，只要这些 run 在原文中连续且斜体属性一致，就不应判为局部斜体错误。Markdown 中的 * 只是报告展示符号，不能当作原文标点或空格来检测。

本 skill 只能辅助注释格式检查，不能保证论文一定符合所有投稿平台、学校或期刊的最终要求。正式投稿前仍应人工复核。

## 常见触发方式
可以用类似下面的方式调用：
帮我检查这篇历史论文的脚注格式。
把这个 Word 文档里的注释按历史研究格式检查一遍。
请生成注释错误报告。
检查这些注释有没有缺作者、期刊名、页码或出版信息。
帮我列出论文注释中不符合历史研究格式的地方。
## 输出文件
默认输出一份文件：
citation-errors.md
## Markdown 文件便于作者直接阅读和修改论文。只有在用户明确要求时，才建议另行生成按具体文献类型设计的 JSON 辅助文件。
