# 历史研究注释格式校正

## 基本介绍

本 skill 用于帮助历史学、近现代史、古代史、专门史、区域史、思想史、制度史等方向的论文写作者，检查脚注、尾注或参考注释是否符合较规范的历史研究注释体例。

它的核心任务不是直接替用户改写 Word 论文，而是识别文献类型、拆解注释字段、检查字段缺漏、检查标点和格式问题，并生成一份可阅读的 Markdown 错误报告。遇到外文书名、期刊名时，应识别并保留斜体信息，在 Markdown 中使用斜体标记，例如 `*Modern China*`；遇到中文文献题名时，应保留书名号《》；遇到文章篇名、析出文献、档案文献、古籍版本等情况时，应按照下列规则处理。

## 什么时候会自动调用

当用户的请求与下列任务有关时，应自动调用本 skill：

1. 检查历史论文、课程论文、毕业论文、学位论文中的脚注或尾注。
2. 将混乱的文献出处改成统一的历史研究注释格式。
3. 校对中文著作、译著、文集析出文献、期刊论文、报纸文章、档案材料、古籍、外文文献等注释。
4. 检查注释中的责任者、题名、出版地、出版社、出版时间、页码等字段是否齐全、顺序是否正确。
5. 保留或修正外文书名、期刊名的斜体标注。
6. 将 Word、Markdown 或纯文本中的注释检查结果输出为 Markdown 错误报告。

常见触发说法包括：

```text
帮我检查这篇历史论文的注释。
把这些脚注改成历史研究格式。
请统一我的论文脚注格式。
帮我检查这些参考注释是否规范。
把中文、英文、档案和古籍注释整理一下。
保留英文书名和期刊名斜体，修改标点和页码格式。
请生成注释错误报告。
```

## 运行逻辑

1. 如果用户提供的是 Word 文档，应先将 `.docx` 解包或用可靠的 Word 解析工具读取正文、脚注、尾注和行内格式。
2. 解析时必须建立两套数据：一套是原始字符流，一套是格式区间。原始字符流必须只包含 Word 中真实存在的字符，包括斜体、空格、全角标点、半角标点、英文逗号、中文逗号、书名号、引号、页码连接号和脚注编号；格式区间单独记录哪些字符是斜体。
3. 不要只抽取纯文本后丢弃格式。外文书名、期刊名是否斜体，是判断注释是否规范的重要依据。
4. 不要把 Markdown 星号当作原文字符。Markdown 的 `*...*` 只能用于最终报告展示，不能参与空格、标点或斜体错误判断。
5. 如果 Word 把同一个斜体词拆成多个 run，但这些 run 连续且斜体属性相同，应合并为同一个斜体区间；不能把 `Eco`、`no`、`mic` 误判成局部斜体错误。
6. 将每条注释拆成可识别字段，并判断其文献类型。
7. 对照下方“注释格式规范”检查字段顺序、字段缺漏、标点、页码和斜体。检查时必须区分首次引证、再次引证、说明性注释和文献类型差异，不要机械要求每条注释都包含完整出版信息。
8. 一次调用必须生成 `citation-errors.md`，而不是只在对话框中展示结果。
9. 除非用户明确要求继续修改原 Word 文档，否则不要直接覆盖或改写用户论文。默认只输出 Markdown 错误报告。

## 输出文件

### citation-errors.md

Markdown 错误报告用于给作者阅读。报告应按注释编号逐条列出问题。报告不能泛泛而谈，必须让作者能够复制报告中的定位片段，在 Word 搜索栏中快速找到问题位置。

每一个问题至少包括：

1. **定位片段**：从原注释中截取一段可复制检索的文字，长度以能唯一定位为宜。
2. **问题对象**：明确指出是哪一个字、哪一个标点、哪一段文字、哪一个空格或哪一个斜体范围有问题。
3. **问题说明**：说明为什么它可能不符合历史研究注释格式。
4. **建议改法**：给出可操作建议；如果无法确定真实文献信息，只说明需要核对或补充，不要编造。

```markdown
## 注释 1

原文：
> 原始注释内容

识别类型：期刊论文

问题 1：英文逗号后缺少空格

- 定位片段：`Minawi,“Telegraphs`
- 问题对象：`Minawi,` 后面紧接左引号 `“`
- 问题说明：英文逗号后通常应有一个半角空格。
- 建议改法：改为 `Minawi, “Telegraphs`

问题 2：外文期刊名斜体范围疑似错误

- 定位片段：`The Geographical Journal`
- 问题对象：原 Word 中 `The Geographical Journal` 的斜体格式区间。
- 问题说明：外文期刊名应整体斜体，不应只让局部文字或引号进入斜体。判断必须来自 Word run 格式区间，不能来自 Markdown 星号。
- 建议改法：回到 Word 原脚注，将期刊名整体设为斜体，文章题名不设斜体。
```

报告只列出问题和建议，不要虚构真实 bibliographic 信息。

### 检测报告的精度要求

1. 指出标点问题时，必须写清楚是哪一个标点，例如“`Minawi,` 后缺少空格”或“`London :` 中冒号前多出空格”。
2. 指出中英文标点问题时，必须写清楚具体字符，例如“这里使用了中文冒号 `：`，英文出版地后应使用英文冒号 `:`”。
3. 指出空格问题时，必须写清楚是“缺少空格”还是“多出空格”，并给出原片段和建议片段。
4. 指出斜体问题时，必须写清楚哪一段文字应该斜体、哪一段文字不应该斜体，不能只写“斜体错误”。
5. 对于 Word 文档，必须利用 run 级格式判断斜体。Markdown 转换后出现的 `**no**`、`Parliamentary** Papers` 等局部星号不是原文字符，不能直接判为原文错误。
6. 判断空格和标点时，必须使用原始字符流。不能因为在报告中插入 Markdown 斜体星号，导致把真实的 `, ` 误判为 `,*`。
7. 判断斜体时，先合并连续相邻且斜体属性相同的 run，再看完整格式区间是否覆盖书名、期刊名或文集名。
8. 只有在原始 Word 格式区间确实显示某段题名中间出现非斜体字符，或题名外部字符被错误设为斜体时，才报告斜体问题。
9. 对缺失字段的判断必须谨慎。再次引证时可以省略出版地、出版社、年份等出版信息；不要把再次引证误判成首次引证缺项。
10. 有些档案、议会文书、古籍或未刊文献本身不一定有卷名、期号或标准出版项；只有在该文献类型按规范应有该字段时，才标记缺失。
11. 如果一条注释由多个文献组成，应尽量按分号拆分后分别判断，并在定位片段中标明是哪一个子文献存在问题。

### 可选的 JSON 辅助结构

默认不输出 `citation-elements.json`。如果用户明确要求生成 JSON 辅助数据，只能按文献类型分别设计字段，不要把著作、期刊、报纸、档案、古籍和外文文献的字段全部混在同一个模板里。

以下仅以“中文期刊文献”为例，其他文献类型应依其自身标注顺序以此类推：

```json
{
  "note_id": "脚注或尾注编号",
  "source_type": "中文期刊文献",
  "original_text": "原始注释文本，尽量保留原标点、空格和页码写法",
  "fields": {
    "author": "",
    "article_title": "",
    "journal_title": "",
    "year": "",
    "issue": "",
    "page": ""
  },
  "formatting": {
    "punctuation_profile": "记录中文逗号、英文逗号、冒号、书名号等使用情况",
    "spaces_preserved": true
  },
  "missing_fields": [],
  "detected_errors": []
}
```

例如档案文献应围绕文献标题、形成时间、卷宗号或其他编号、藏所设计字段；古籍应围绕责任者、文献题名、卷次、篇名、版本和页码设计字段；外文文献应另外记录外文书名或期刊名的斜体状态。

## 使用方法

1. 先判断每条注释属于哪一种文献类型：著作、译著、析出文献、序言、古籍、期刊、报纸、档案、外文文献、再次引证、间接引文或夹注。
2. 按照相应类型的“标注顺序”检查字段。
3. 保留原文已有的有效信息，不要凭空编造作者、出版社、出版年、页码、卷册号或馆藏信息。
4. 如果出版时间缺失，按照规范写作 `[出版时间不详]`；如果其他必要信息缺失，应明确提示用户“缺某项信息”，不要假装已经补齐。
5. 中文文献题名使用《》；外文文章题名使用英文引号；外文书名、期刊名使用 Markdown 斜体 `*书名或刊名*`。
6. 注意标点：中文注释中的逗号、冒号、书名号和句号应与示例一致；英文注释中的逗号、卷号、期号、出版地、出版社、年份和页码标识应与示例一致。
7. 页码一般放在注释末尾。中文文献使用“第x页”“第x-y页”；英文文献使用 `p.` 或 `pp.`。
8. 输出 `citation-errors.md`，并在对话中简要说明文件位置和主要发现。只有在用户明确要求 JSON 辅助数据时，才另行生成按文献类型定制的 JSON 文件。

## 再次引证与弹性判断

1. 同一文献再次引证时，通常只需保留责任者、题名和页码，出版地、出版社、年份等出版信息可以省略。
2. 如果注释只写作者、短题名和页码，应先判断它是否可能是再次引证；只有在无法对应前文完整引证时，才提示“需确认前文是否已有完整引证”。
3. 不同文献类型的必要字段不同。不要要求档案文献必须有出版社，也不要要求议会文书、古籍、未刊手稿一定具备期刊卷号或期号。
4. 对疑似问题使用“疑似”“需复核”“建议确认”这样的措辞；对确定问题使用“缺少”“多出”“应改为”这样的措辞。
5. 检测报告的目标是帮助作者定位和修改，不是一次性给出绝对判决。

## 能力边界

1. 使用前应确认用户的论文、期刊、学校或出版社确实要求采用历史研究注释格式。不同机构可能有自己的格式细则。
2. 如果原注释缺少期刊名、刊号、卷号、期号、页码、出版社、出版时间、档案编号或馆藏地点，本 skill 不能自动补全，只能标记缺失。
3. 默认情况下，本 skill 不自动改写用户 Word 文档中的注释，只生成 Markdown 错误报告。
4. 用户如果希望继续修正文档，应根据错误报告继续用自然语言要求 Codex、编程工具或第三方平台修改论文。
5. 不要为了让格式“看起来完整”而编造来源信息。

## 注释格式规范

## 一、 注释体例及标注位置

文献引证方式采用注释体例。

注释放置于当页下（脚注）。注释序号用①，②，③……标识，每页单独排序。正文中的注释序号统一置于包含引文的句子(有时候也可能是词或词组)或段落标点符号之后。

## 二、注释的标注格式

### （一）非连续出版物

#### 1．著作

标注顺序:责任者与责任方式/文献题名/出版地点/出版者/出版时间/页码。

责任方式为著时，“著”可省略，其他责任方式不可省略。

引用翻译著作时，将译者作为第二责任者置于文献题名之后。

引用《马克思恩格斯全集》、《列宁全集》等经典著作应使用最新版本。

**示例：**

赵景深：《文坛忆旧》，上海：北新书局，1948年，第43页。

谢兴尧整理：《荣庆日记》,西安：西北大学出版社，1986年，第175页。

蒋大兴：《公司法的展开与评判——方法·判例·制度》，北京：法律出版社，2001年，第3页。

任继愈主编：《中国哲学发展史(先秦卷)》,北京：人民出版社,1983年,第25页。

实藤惠秀：《中国人留学日本史》，谭汝谦、林启彦译，香港：中文大学出版社，1982年，第11-12页。

金冲及主编：《周恩来传》，北京：人民出版社、中央文献出版社，1989年，第9页。佚名：《晚清洋务运动事类汇钞五十七种》上册，北京：全国图书馆文献缩微复制中心，1998年，第56页。

狄葆贤：《平等阁笔记》，上海：有正书局，[出版时间不详]，第8页。

《马克思恩格斯全集》第31卷，北京：人民出版社，1998年，第46页。

#### 2．析出文献

标注顺序:责任者/析出文献题名/文集责任者与责任方式/文集题名/出版地点/出版者/出版时间/页码。

文集责任者与析出文献责任者相同时，可省去文集责任者。

**示例：**

杜威·佛克马:《走向新世界主义》,王宁、薛晓源编：《全球化与后殖民批评》,北京：中央编译出版社,1999年,第247-266页。

鲁迅：《中国小说的历史的变迁》，《鲁迅全集》第9册，北京：人民文学出版社，1981年，第325页。

唐振常：《师承与变法》，《识史集》，上海：上海古籍出版社，1997年，第65页。

#### 3.著作、文集的序言、引论、前言、后记

（1）序言、前言作者与著作、文集责任者相同。

**示例：**

李鹏程：《当代文化哲学沉思》，北京：人民出版社，1994年，“序言”，第1页。

（2）序言有单独的标题，可作为析出文献来标注。

**示例：**

楼适夷：《读家书，想傅雷（代序）》，傅敏编：《傅雷家书》（增补本），北京：三联书店，1988年，第2页。

黄仁宇：《为什么称为“中国大历史”？——中文版自序》，《中国大历史》，北京：三联书店，1997年，第2页。

（3）责任者层次关系复杂时，可以通过叙述表明对序言的引证。为了表述紧凑和语气连贯，责任者与文献题名之间的冒号可省去，出版信息可括注起来。

**示例：**

见戴逸为北京市宣武区档案馆编、王灿炽纂《北京安徽会馆志稿》（北京：北京燕山出版社，2001年）所作的序，第2页。

#### 4.古籍

（1）刻本

标注顺序：责任者与责任方式／文献题名（卷次、篇名、部类）（选项）／版本、页码。

部类名及篇名用书名号表示，其中不同层次可用中圆点隔开，原序号仍用汉字数字，下同。页码应注明a、b面。

**示例：**

姚际恒：《古今伪书考》卷3，光绪三年苏州文学山房活字本，第9页a。

（2）点校本、整理本

标注顺序：责任者与责任方式／文献题名／卷次、篇名、部类（选项）／出版地点／出版者／出版时间／页码。可在出版时间后注明“标点本”“整理本”。

**示例：**

毛祥麟：《墨余录》，上海：上海古籍出版社，1985年，第35页。

（3）影印本

标注顺序：责任者与责任方式／文献题名／卷次、篇名、部类（选项）／出版地点／出版者／出版时间／（影印）页码。可在出版时间后注明“影印本”。为便于读者查找，缩印的古籍，引用页码还可标明上、中、下栏（选项）。

**示例：**

杨钟羲：《雪桥诗话续集》卷5，沈阳：辽沈书社，1991年影印本，上册，第461页下栏。

《太平御览》卷690《服章部七》引《魏台访议》，北京:中华书局,1985年影印本，第3册，第3080页下栏。

（4）析出文献

标注顺序：责任者／析出文献题名／文集责任者与责任方式／文集题名／卷次／丛书项（选项，丛书名用书名号）／版本或出版信息／页码。

**示例：**

管志道：《答屠仪部赤水丈书》，《续问辨牍》卷2，《四库全书存目丛书》，济南：齐鲁书社，1997年影印本，子部，第88册，第73页。

（5）地方志

唐宋时期的地方志多系私人著作，可标注作者；明清以后的地方志一般不标注作者，书名其前冠以修纂成书时的年代（年号）；民国地方志，在书名前冠加“民国”二字。新影印（缩印）的地方志可采用新页码。

**示例：**

乾隆《嘉定县志》卷12《风俗》，第7页b。

民国《上海县续志》卷1《疆域》，第10页b。

### （三）未刊文献

#### 1.学位论文、会议论文等

### （四）转引文献

无法直接引用的文献，转引自他人著作时，须标明。标注顺序：责任者／原文献题名／原文献版本信息／原页码（或卷期）／转引文献责任者／转引文献题名／版本信息／页码。

**示例：**

### （五）电子文献

电子文献包括以数码方式记录的所有文献（含以胶片、磁带等介质记录的电影、录影、录音等音像文献）。

标注项目与顺序：责任者／电子文献题名／更新或修改日期／获取和访问路径／引用日期。

**示例：**

王明亮：《关于中国学术期刊标准化数据库系统工程的进展》，1998年8月16日， http://www.cajcd.cn/pub/wml.txt/980810-2.html, 1998年10月4日。

#### 1.引证外文文献，原则上使用该语种通行的引证标注方式。

## 三、其他

### （一）再次引证时的项目简化

同一文献再次引证时只需标注责任者、题名、页码，出版信息可以省略。

**示例：**

赵景深：《文坛忆旧》，第24页。

鲁迅：《中国小说的历史的变迁》，《鲁迅全集》第9册，第326页。

### （二）间接引文的标注

间接引文通常以“参见”或“详见”等引领词引导，反映出与正文行文的呼应，标注时应注出具体参考引证的起止页码或章节。标注项目、顺序与格式同直接引文。

**示例：**

参见邱陵编著：《书籍装帧艺术简史》，哈尔滨：黑龙江人民出版社，1984年，第28－29页。

详见张树年主编：《张元济年谱》，北京：商务印书馆，1991年，第6章。

### （三）引用先秦诸子等常用经典古籍，可使用夹注，夹注应使用不同于正文的字体。

**示例1：**

庄子说惠子非常博学，“惠施多方，其书五车。”（《庄子·天下》）

**示例2：**

天神所具有道德，也就是“保民”、“裕民”的道德；天神所具有的道德意志，代表的是人民的意志。这也就是所谓“天聪明自我民聪明，天明畏自我民明畏”（《尚书·皋陶谟》），“民之所欲，天必从之”（《尚书·泰誓》）。
